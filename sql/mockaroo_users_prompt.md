# Mockaroo prompt for `users`

Generate realistic sample data for a MySQL `users` table for a car rental app.

Requirements:

- Create 30 to 100 rows.
- Make most rows regular users and only a few rows admins.
- Set `role` to mostly `user`, sometimes `admin`.
- Use realistic first and last names.
- Make every `email` unique.
- Use realistic phone numbers.
- Set `password_hash` to a placeholder hashed password string.
- Use a recent `created_at` timestamp.
- Keep the data clean and believable.

Output fields:

- `id`
- `role`
- `first_name`
- `last_name`
- `email`
- `phone`
- `password_hash`
- `created_at`
